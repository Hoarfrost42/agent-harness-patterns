# 案例：旧文档污染

## 失败模式

Agent 读到一份旧后端契约，并把它当成当前事实。因为这份契约写得很正式，Agent 又把它写进 handoff。下一个窗口继续按这份契约规划工作，结果面向的是一个已经不存在的 API。

## 更好的 Harness

写 handoff 或 read-first index 前：

1. 对照旧契约和当前代码、运行行为。
2. 搜索已经改名或删除的路由。
3. 检查 package scripts、runbook 和生产行为。
4. 把旧材料分类为历史参考、降级参考、重写为当前事实，或删除候选。

## 规则

写得正式不等于权威。当前代码和运行行为优先于旧文档。
