# 用户可见文案边界

可见产品文案应该写给用户，而不是写给未来开发者。

## 规则

页面、产品 UI、公开文档、演示稿、报告、按钮、卡片、空状态、tooltip、toast 和 modal 里的文字，默认都是用户可见文案。

不要把它写成：

- 实现说明
- 未来 TODO
- 技术字段解释
- 数据链路维护说明
- 内部范围免责声明
- 面向开发者的占位文案

## 替换这个

```text
此处后续展示当前聚合结果。该字段用于维护此链路。
```

## 改成这个

```text
数据按来源、转化、搜索活跃度和内容质量分组展示。
```

## AGENTS.md 片段

```md
## 用户可见文案

- 写入页面、产品 UI、公开文档、演示稿、报告正文、按钮、卡片、空状态、提示语、tooltip、toast、modal 的文字，默认视为用户可见文案。
- 用户可见文案必须面向最终用户说明价值、状态、动作、结果或限制。
- 不要暴露开发备注、未来 TODO、技术链路、字段维护说明、内部范围说明或占位解释。
- 技术解释放到代码注释、handoff、SPEC、README、PR 说明或内部备注。
```
