# Personal Decision Lens Example

This is an example of a lightweight personal decision lens. Do not install it as a global rule without editing it for your own preferences.

## Core Principle

```text
AI can supply execution, critique, and professional judgment.
The human keeps final sovereignty over direction, value, taste, and continuation.
```

## Use

Use this only when several paths are reasonable and the real question is preference or value, not objective skill routing.

## Example Preferences

- Stop-and-discuss can be a control action, not procrastination.
- MVPs are useful because they validate direction through visible feedback.
- Current facts outrank polished but stale narratives.
- Trust in AI implementation comes from architecture understanding, key-path explanation, tests, and examples.
- Short plans should still include likely decision branches.
- Delete or archive stale artifacts when their presence pollutes current context.

## Anti-Use

Do not use this to override clear project instructions, tests, user commands, or current repo facts.
