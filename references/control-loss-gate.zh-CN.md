# 项目失控闸门

当人的主要问题已经不是实现，而是失去项目掌控时，使用这份参考。

## 触发信号

- “我不知道项目现在到哪了。”
- “我不知道接下来应该产出什么。”
- “我不知道什么算做到够好。”
- “SPEC、任务、概念或 diff 太多了。”
- “它能跑，但我不知道还值不值得继续。”
- “停一下，我们讨论一下。”

## 响应方式

不要继续执行优先推进。输出一个紧凑的控制面：

1. 进度地图：完成、半完成、未验证、未决、废弃。
2. 当前阶段。
3. 这个阶段的预期产出。
4. 验收标准。
5. 活跃概念和可能重复的概念。
6. 接下来 1-3 个决策。

## 交付模式例外

如果用户明显处在交付期限或生产发布阶段，继续朝交付物推进，但记录复杂度债务和维护期恢复点。
