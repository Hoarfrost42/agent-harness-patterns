# Adoption Levels

Use the smallest level that solves the current problem.

## Level 1: Paste A Rule

Best for people who do not want a skill system yet.

Start with:

- `snippets/command-output.md`
- `snippets/user-visible-copy.md`
- `snippets/context-hygiene.md`

Expected value: fewer context blowups, less developer-facing UI copy, clearer local instructions.

## Level 2: Use A Reference

Best when the problem is judgment, not implementation.

Start with:

- `references/control-loss-gate.md`
- `references/skills-index-template.md`
- `references/personal-decision-lens-example.md`

Expected value: better stop/go decisions, clearer read-first order, fewer accidental duplicate workflows.

## Level 3: Install One Skill

Best when the same failure mode keeps repeating.

Start with:

- `skills/project-continuity-control`
- `skills/straight-line-code-discipline`
- `skills/no-visible-meta-copy`

Expected value: reusable behavior across windows and projects.

## Level 4: Build Your Own Harness

Best when you already know your repeated failure modes.

Do this only after a few real tasks:

1. Write down the failure mode.
2. Capture the smallest rule that prevented it.
3. Test it in another task.
4. Keep it only if it reduces rework or restores control.

## Rule

Do not upgrade levels because the system looks elegant. Upgrade only when a real task repeats the same pain.
